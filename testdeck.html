Steps for Go Live in January 1. Update all Automated email templates with appropriate messages for each email interval. (Don't use {{{Sender.Signature}}} in the email templates. Slide 33: Web to Lead Proof of Concept Testing

In Setup > Web to Lead and “Create Web-to-Lead” form For Web to Lead Proof of Concept Testing ● Correct action attribute: Your form's action attribute must be set to the Salesforce endpoint: https://test.salesforce.com/servlet/servlet.WebToLead?encoding=UTF-8. If you are using a production environment, the URL will be https://webto.salesforce.com/servlet/servlet.WebToLead?encoding=UTF-8. ● Essential Hidden Fields: The Salesforce form generator includes specific hidden input fields that are essential for the submission to work correctly. You must copy these from the generated code into your existing form: ● oid: This is your Salesforce Organization ID, which tells Salesforce where to create the lead. ● retURL: This specifies the URL the user will be redirected to after the form is successfully submitted. ● Matching Field Names: The name attributes of your existing form's input fields (e.g., for First Name, Last Name, Email) must precisely match the name attributes in the Salesforce-generated HTML (e.g., first_name, last_name, email). ● Required Fields: Ensure all fields that are marked as "required" in your Salesforce Lead object settings or page layouts are included in your website form and have data submitted. Submissions that violate validation rules will fail to create a lead record.
