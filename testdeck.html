Organization-Wide Email Addresses & DKIM

Users/Automations can use Organization-Wide Email Addresses to send messages from generic emails.

  1. Setup → Organization-Wide Addresses
  2. Allow either All or Selected Profiles to each Organization-Wide Address
Once you click Save, the user managing the email account will receive a confirmation email and will need to click on the link to activate this process.

A DKIM Key was also configured to minimize external server rejection of un-verified email.

Web to Lead

Functionality could be explored for churchinvestorsfund.org "Invest Now" action. Web to Lead allows you to select fields to be filled, generates an HTML form with a form action endpoint that will create the lead record in Salesforce.

For Web to Lead Proof of Concept Testing:

  1. Correct action attribute: Your form's action attribute must be set to the Salesforce endpoint. For sandbox: https://test.salesforce.com/servlet/servlet.WebToLead. For production: https://webto.salesforce.com/servlet/servlet.WebToLead
  2. Essential Hidden Fields: Include oid (Organization ID) and retURL (redirect URL after submission)
  3. Matching Field Names: The name attributes must precisely match Salesforce-generated HTML (e.g., first_name, last_name, email)
  4. Required Fields: Ensure all required fields in your Salesforce Lead object are included. Submissions that violate validation rules will fail.
