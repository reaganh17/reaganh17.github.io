Web to Lead [cite: 192]

[cite_start]

Functionality could be explored for https://churchinvestorsfund.org/ “Invest Now” action. [cite: 192] [cite_start]Web to Lead allows you to select fields, generates an HTML form with a form action endpoint that will create the lead record in salesforce. [cite: 193]

[cite_start]

Since the webform is already fully built out on churchInvestorsfund.org, your website team might be able to leverage the action instead of using Salesforce’s form. [cite: 194] [cite_start]To create the form, go to **Setup > Web to Lead** and “Create Web-to-Lead” form. [cite: 195]

[cite_start]

Web to Lead Proof of Concept Testing [cite: 196]

    [cite_start]
  • **Correct action attribute**: Your form's action attribute must be set to the Salesforce endpoint: https://test.salesforce.com/servlet/servlet.WebToLead?encoding=UTF-8 (or https://webto.salesforce.com/servlet/servlet.WebToLead?encoding=UTF-8 for production). [cite: 196, 197]
    • [cite_start]
  • **Essential Hidden Fields**: You must copy the following hidden input fields from the generated code into your existing form: [cite: 198, 199]
      [cite_start]
    • oid: Your Salesforce Organization ID. [cite: 199]
      • [cite_start]
    • retURL: The URL the user is redirected to after successful submission. [cite: 200]
    • [cite_start]
  • **Matching Field Names**: The name attributes of your existing form's input fields (e.g., for First Name, Last Name, Email) must precisely match the name attributes in the Salesforce-generated HTML (e.g., first_name, last_name, email). [cite: 201]
    • [cite_start]
  • **Required Fields**: Ensure all fields marked as "required" in your Salesforce Lead object settings or page layouts are included in your website form and have data submitted, as submissions violating validation rules will fail. [cite: 202, 203]
