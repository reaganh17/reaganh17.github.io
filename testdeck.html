Go Live Checklist

Steps for Go Live in January

  1. Update all Automated email templates with appropriate messages for each email interval.
    (Don't use {{{Sender.Signature}}} in the email templates. The logo will not render.)
  2. Update Church Visit Cadence with the email templates and intervals that make sense for your processes.
  3. Ensure the "Add to email thread" checkbox is selected or deselected based on which option looks better after testing.
  4. Note: if you'd like the Investments Team and Loans Team to be able to see each other's Cadences, update the Cadence Folder Sharing (Same goes for the Email Template Folder Sharing).
  5. Remove Sales Engagement Basic User Permissions and Assign Sales Engagement User Permission to Users (having both on one user can cause issues).
  6. Users can set Cadence Settings working days and hours in their Personal Information settings (from view profile Avatar on the upper right of the page).

Email Configuration (DKIM)

Organization-Wide Email Addresses & DKIM

Users/Automations can use Organization-Wide Email Addresses to send messages from generic emails: **Setup > Organization-Wide Addresses**. You must Allow either **All** or **Selected Profiles** to each Organization-Wide Address.

**NOTE:** Once you click Save, the user managing the email account will receive a confirmation email and will need to click on the link to activate this process.

A **DKIM Key** was also configured to minimize external server rejection of un-verified email.

Outlook Integration

The Outlook Integration allows users to **Log emails to Salesforce records** and **Create Salesforce records from Outlook**.

  • The default Inbox Email Pane for Outlook is set via **Setup > Outlook Integration and Sync**.

Leads and Automations

Lead Process - Church Visit

The Lead record type is **Lead**. Future considerations include Lead Assignment Rules, Web to Lead, and Lead Auto Response Rules.

Note that **Lead** has not been added as a Navigation Item to any App.

Automations (Flows)

  • **Update Lead Status Investments Cadence**: This flow updates the Lead Status field as the Lead continues through the Investments Cadence. After the first contact, the Lead is set to **Contacted**. After the first automated email at the two-week mark, the Lead is set to **Nurturing** and stays in this status.
  • **Remove Lead From Cadence**: When a lead's status is set to "Unqualified", they are removed from the Cadence.
  • **Update Opportunity Fields on Lead Convert**: This flow looks for isConverted = true on the Lead object and Updates the corresponding Opportunity fields.
  • **Request DKIM Access Flow**: Screenflow Starts a session for DKIM key access.
Various Sales Engagement Flow Actions are available, such as Assign Target to Cadence, Modify Cadence Tracker Attributes, and Remove Target from Cadence.

Web to Lead

Functionality could be explored for https://churchinvestorsfund.org/ “Invest Now” action. Web to Lead allows you to select fields, generates an HTML form with a form action endpoint that will create the lead record in salesforce.

Since the webform is already fully built out on churchInvestorsfund.org, your website team might be able to leverage the action instead of using Salesforce’s form. To create the form, go to **Setup > Web to Lead** and “Create Web-to-Lead” form.

Flows for Mass Emails

If you need to send a generic reminder email to all or a portion of your partners, you can create a Flow to do the work for you. Choose either a **screen flow** or a **scheduled flow** depending on your user’s preference.
